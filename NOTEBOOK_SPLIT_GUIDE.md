# Notebook Split Documentation

## Overview
The original `llm_behavior_analysis.ipynb` has been separated into two focused notebooks to improve organization and clarity:

### Part 1: llm_behavior_analysis.ipynb
**Purpose:** Behavior Analysis & LLM Processing  
**Sections:** 1-7 (19 cells)

Contains:
- **SECTION 1:** Configuration & Infrastructure
- **SECTION 2:** Load Behavior IR & Deterministic Aggregation
- **SECTION 3:** IR-to-Prompt Serializer
- **SECTION 4:** Prompt Templates
- **SECTION 5:** LLM Client
- **SECTION 6:** Per-Sample LLM Analysis (Cached)
- **SECTION 7:** Family-Level LLM Analysis (Cached)

**Outputs:**
- Cached LLM analysis results
- Per-sample semantic interpretations
- Family-level behavior patterns
- Ready-to-use data for signature generation

---

### Part 2: signature_generation_validation.ipynb
**Purpose:** Signature Generation & Validation  
**Sections:** 8-10 (9 cells)

Contains:
- **SECTION 8:** CAPEv2 Signature Generation (Cached)
- **SECTION 9:** Save All Outputs
- **SECTION 10:** Validation & Visualization

**Outputs:**
- CAPEv2 Python signatures
- Signature validation reports
- MITRE ATT&CK mappings
- Comprehensive visualizations

---

## Workflow

### Step 1: Run Part 1
Execute `llm_behavior_analysis.ipynb` to:
- Load and aggregate Behavior IR data
- Generate LLM-based semantic analyses
- Create family-level interpretations
- Cache all results for reuse

### Step 2: Run Part 2
Execute `signature_generation_validation.ipynb` to:
- Generate CAPEv2 Python signatures from cached LLM analyses
- Validate signature quality
- Map techniques to MITRE ATT&CK framework
- Generate comprehensive reports and visualizations

## Running Both Notebooks

```
1. Open llm_behavior_analysis.ipynb
2. Run all cells (Shift+Ctrl+Enter or via Run menu)
3. Wait for completion (check cache_stats() output)
4. Open signature_generation_validation.ipynb
5. Run all cells
```

The LLM cache ensures instant resumption and prevents redundant API calls.
